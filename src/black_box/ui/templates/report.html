Findings

The operator's hypothesis blames the tunnel at t=02:57:11. The recording shows otherwise. RTK fix quality collapsed E1 at t=02:14:32, forty-three minutes before tunnel entry, while the robot was still in open lot with clear sky E3. For the ninety seconds preceding that collapse, the EKF heading covariance silently widened from 0.018 to 0.21 rad E2 without triggering /diagnostics_agg; a single rate-limited warning E5 was the only signal, and it was suppressed. A secondary GNSS receiver was already disagreeing with the primary by more than 0.40m for 280 seconds prior E8, but no code path consumed that signal E6.

By the time the robot reached the tunnel E4, it was already operating on a degraded estimator that self-reported nominal. The tunnel revealed the fault. It did not cause it.
